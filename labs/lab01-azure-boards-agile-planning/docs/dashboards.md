# Dashboard Reporting

## Objective

Create Azure DevOps dashboards to visualize project health, sprint status, and workload distribution.

## Dashboard Created

Product Training Dashboard

The dashboard was configured for the EShop-Web team.

## Widgets Added

### Sprint Overview

Provides visibility into:

* Sprint progress
* Work item status
* Team delivery metrics

### Sprint Capacity

Displays:

* Available capacity
* Assigned work
* Remaining workload

### Work Item Chart

A pie chart was generated from a shared query.

Query:

Web Tasks

Grouping:

Assigned To

Purpose:

Visualize task ownership and workload distribution.

## Benefits

* Centralized project visibility
* Faster decision making
* Improved sprint monitoring
* Better workload transparency

## Outcome

A customized dashboard was created to provide stakeholders and team members with real-time project insights.
