# Sprint Management and Capacity Planning

## Objective

Configure sprint planning, team capacity, workload distribution, and sprint tracking using Azure DevOps Boards.

## Sprint Configuration

The project was organized into:

* Sprint 1
* Sprint 2
* Sprint 3

Each sprint was configured using Scrum methodology and time-boxed delivery.

## Capacity Planning

Team member capacity was configured using:

* Activity Type: Development
* Capacity Per Day: 1 hour
* Planned Days Off

This configuration allowed Azure DevOps to calculate available sprint capacity automatically.

## Task Assignment

Tasks were assigned and estimated using Remaining Work values.

Examples:

* Add page for most recent tutorials
* Optimize data query for most recent tutorials

## Workload Analysis

Azure DevOps highlighted situations where sprint workload exceeded available team capacity.

Benefits include:

* Improved sprint forecasting
* Better workload balancing
* Reduced risk of sprint overcommitment

## Board Customization

Taskboard settings were customized to improve visibility by:

* Assigned To
* Activity Type
* Styling Rules

## Outcome

Sprint planning and capacity management were successfully configured, allowing the team to monitor workload, assignments, and delivery expectations throughout the sprint lifecycle.
