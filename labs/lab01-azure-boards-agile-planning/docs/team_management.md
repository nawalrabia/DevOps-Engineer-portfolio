# Team Management and Sprint Configuration

## Objective

Configure Azure DevOps teams, area paths, and sprint iterations to support Agile project management.

## Team Configuration

A dedicated team named **EShop-Web** was created within the Azure DevOps project.

### Benefits

* Team-specific backlog management
* Sprint planning
* Capacity tracking
* Dashboard customization

## Iteration Configuration

Three sprint iterations were configured:

* Sprint 1
* Sprint 2
* Sprint 3

These iterations support Agile delivery using time-boxed development cycles.

## Area Path Configuration

The team area path was configured to include sub-areas.

### Benefits

* Visibility across multiple work streams
* Better portfolio management
* Improved reporting capabilities

## Outcome

The Agile team structure was successfully configured and prepared for backlog and sprint management activities.
