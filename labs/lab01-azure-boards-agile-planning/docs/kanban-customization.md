# Kanban Board Customization

## Objective

Customize Azure DevOps Kanban boards to improve workflow visualization, work tracking, and team collaboration.

## Tag Configuration

Custom tags were created to classify work items.

Examples:

* data
* ux

Tag colors were configured to improve visibility on the board.

## Workflow Customization

A custom workflow stage was introduced:

* QA Approved

This stage was placed between:

* Committed
* Done

## Work In Progress (WIP) Limits

A WIP limit was configured for the QA Approved stage.

Benefits:

* Prevent bottlenecks
* Improve flow efficiency
* Identify overloaded stages

## Split Columns

The QA Approved stage was split into:

* Doing
* Done

This provided better visibility into actual work progress.

## Definition of Done

Definition of Done:

Passes all tests

This ensures quality requirements are met before work progresses.

## Swimlanes

An Expedite swimlane was created for high-priority work items.

Benefits:

* Prioritize urgent work
* Improve visibility
* Support faster delivery

## Outcome

The Kanban board was successfully customized to support the team's workflow, quality controls, and prioritization strategy.
