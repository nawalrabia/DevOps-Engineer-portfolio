# Work Item Management

## Objective

Manage project requirements using Azure DevOps work item hierarchy.

## Work Item Structure

The project was organized using Scrum work item relationships.

### Epic

Product Training

### Feature

Training Dashboard

### Product Backlog Items

* As a customer, I want to view new tutorials
* As a customer, I want to see tutorials I recently viewed
* As a customer, I want to request new tutorials

### Tasks

* Add page for most recent tutorials
* Optimize data query for most recent tutorials

## Benefits

Using a hierarchical work item structure provides:

* Requirement traceability
* Better sprint planning
* Improved backlog management
* Progress tracking from business goals to implementation tasks

## Outcome

The project backlog was successfully decomposed from Epic level down to implementation tasks.
